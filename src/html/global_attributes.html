グローバル属性

    • 全てのHTML要素に使用可能
    • タグによって効果がないこともある

    accesskey

    • accesskey属性はショットカットキーを指定する属性
    • ブラウザとOSの組み合わせによってショットカットキーの起動の仕方は異なる
      accesskey--MDN

    このボタンタグにはaccesskey="s"を指定している

    autocapitalize

    • 物理的なキーボード（パソコンの）では影響がない。
      音声入力・モバイル・仮想キーボードの時だけ動く
    • typeがurl, email, passwordのinputタグにautocapitalize属性を追加しても動かない
    • 入力・修正するテキストの英文字の大文字・小文字を制御する
    • off or none 全ての文字小文字に
    • on or sentence 文の頭文字だけ大文字、他の文字小文字
    • words 単語ごと頭文字を大文字に
    • characters 全ての文字を大文字に

    contenteditable

    • ユーザーによる編集が可能であることを指定できる属性
    • true or false

    このテキストはcontenteditable属性がfalseである

    • inert

    inputmode

    • input typeと一緒に使えばいい
    • モバイルで確認できた（仮想キーボード）
    • none: キーボードが出ない
    • text: 普通
    • tel: 電話の時のキーボード
    • url: textの,が/に変わる。enterkeyhintも（enterキーの模様）変わる
    • email: textの,が＠に変わる。enterkeyhintはurlと一緒の模様になる
    • numeric: telは数字の後ろにアルファベットが一緒にあったが、これは数字だけでスッキリ
    • decimal: numericと見た目一緒
    • search: textと見た目一緒

    is

    • カスタムタグ作った時名前指定してその名前をタグに使うじゃ
      名前変えずにis属性にjs側でdefineしたキーワード入れれば同じ動きする
