    selector

    優先順位

    1. color: red !important;
    2. <someTag style="color: red; back..."
    3. #idName { ... }
    4. .className { ... }, selector[attr="value"] { ... }
    5. h1::first-letter { ... }
    6. body, h1, p { ... }
    7. *, +, >, ~, :not()は優先順位に影響がない。

    属性セレクター

    • a[title] title属性があるaタグ
    • a[title="value"] valueと完全一致のみ
    • a[title~="value"] valueと完全一致 or 空白の区切りでvalueと一致している
      <a title="valueX value" OK
      <a title="value valueX" OK
      <a title="value/valueX" NG
    • a[title|="value"] valueと完全一致 or valueの後ろにハイフン（-）がつくもの
      <a title="value-valueX" OK
      <a title="valueX-value" NG
      <a title="valueY-value-valueX" NG
    • a[title^="va"] titleがvaで始まるaタグ
    • a[title$="ue"] titleがueで終わるaタグ
    • a[title*="xyz"] titleの中にxyzを含んでいるaタグ
    • セレクターの最後に「i」を入れたら大文字小文字の関係なく選択できる
      a[title*="abc" i]

    combinator

      • .className div = .classNameの中にあるpタグの全部
      • .className > div = .classNameのすぐ下にあるpタグだけ (そのdivタグの中にdivタグがあったとしてもそれは反映されない)(1段階見る)
    • h1 + p = h1タグのすぐ後にくるpタグ１個(1段階見る)。中・前にあっても無効。h1タグとpタグの間他の要素があったら適応されない
    • h1 ~ p h1タグの後ろにあるpタグ(1段階見る)

    特性・点数・優先順位

    • /* specificity: 0-3-3 */
      div div .nav:nth-child(2) a:hover {
      border: 10px double black;
      }

    • div, div, aで0-0-3点
    • .nav :nth-child() :hover　で0-３-0点
    • .navがliタグの場合

      /* specificity: 0-2-4 */
      div div li:nth-child(2) a:hover {
      border: 5px solid red;
      }
      これはliを使っているために0-3-3より後ろに書いたとしても負けて適応されない

    • id-class, pseudo-tag　の点数構成（0-0-0）だけどインラインスタイルには勝てない(0-0-0-0)
    • 全部勝てるのが!important　必要な時じゃなければ絶対使わないほうがいい　cascadeの作動方法を変更させる
